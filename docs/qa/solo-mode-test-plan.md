## 1. 測試範圍

本測試計畫涵蓋 Betrayal at House on the Hill 單人模式的完整功能驗證，包含角色選擇、遊戲版圖、回合流程、房間發現、卡牌系統及作祟系統。

### 1.1 測試目標

- 驗證所有單人模式功能符合規則書規定
- 確保遊戲邏輯正確執行
- 建立可追溯的測試證據
- 提供回歸測試基準

### 1.2 參考文件

| 文件 | 路徑 |
|------|------|
| 規則書摘要 | `shared-data/rulebook/rulebook-summary.md` |
| 角色資料 | `shared-data/characters/characters.ts` |
| 房間資料 | `shared-data/rooms/rooms.ts` |
| 卡牌資料 | `shared-data/cards/README.md` |

## 4. 測試執行計畫

### 4.1 執行順序

```
Phase 1: 基礎功能測試
├── TC-CHAR-001 ~ 004 (角色選擇)
└── TC-BOARD-001 ~ 002 (地圖渲染與移動)

Phase 2: 核心機制測試
├── TC-TURN-001 ~ 004 (回合流程)
├── TC-DISC-001 ~ 004 (房間發現)
└── TC-CARD-001 ~ 004 (卡牌系統)

Phase 3: 進階功能測試
├── TC-HAUNT-001 ~ 004 (作祟系統)
└── TC-BOARD-003 ~ 004 (進階地圖功能)
```

### 4.2 相依性說明

| 測試案例 | 依賴 | 說明 |
|----------|------|------|
| TC-BOARD-002 | TC-CHAR-004 | 需要角色選擇完成後才能移動 |
| TC-TURN-001 | TC-BOARD-002 | 需要能移動才能測試回合流程 |
| TC-DISC-001 | TC-TURN-002 | 需要移動功能正常 |
| TC-CARD-001 | TC-DISC-001 | 需要能發現房間才能進入符號房間 |
| TC-HAUNT-001 | TC-CARD-003 | 需要能抽預兆卡才能檢定 |

### 4.3 測試環境需求

- **瀏覽器:** Chrome 120+, Firefox 121+, Safari 17+
- **解析度:** 1920x1080 (桌面), 390x844 (手機)
- **遊戲模式:** 單人模式 (Solo Mode)
- **AI 數量:** 2-5 個 AI 玩家

## 6. 缺陷嚴重程度定義

| 等級 | 定義 | 範例 |
|------|------|------|
| **P0 - 致命** | 遊戲無法進行或主要功能完全失效 | 無法選擇角色、回合無法結束 |
| **P1 - 嚴重** | 主要功能受損，有替代方案 | 房間不旋轉、卡牌效果錯誤 |
| **P2 - 中等** | 次要功能問題 | UI 顯示錯誤、動畫問題 |
| **P3 - 輕微** | 外觀或建議性問題 | 文字錯誤、顏色不一致 |
